<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Public Internet -> ngrok tunnel | External users access the attack panel via ngrok HTTPS URL; traffic is encrypted by ngrok but the URL is discoverable |
| ngrok tunnel -> Streamlit attack panel (localhost:8502) | Decrypted traffic reaches the local Streamlit app; ngrok is the trust gateway |
| Attack panel -> Docker nginx (localhost:8080) | Attack HTTP requests are sent from the attack panel to the local nginx container; this is a trusted local path |
| Attack panel -> AttackSimulator worker threads | The Streamlit app spawns threads that send HTTP requests; these run as the same OS user |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | ngrok URL | accept | ngrok free tier provides random subdomains. URL sharing is limited to classroom. No sensitive data exposed -- this is a demo environment. |
| T-01-02 | Tampering | Attack payloads | mitigate | Attack payloads are hardcoded in src/attack_simulator.py (SQL_INJECTION_PAYLOADS list). The attack panel only allows selecting from predefined types -- user cannot inject arbitrary HTTP requests or target external servers. The base_url is hardcoded to localhost:NGINX_PORT. |
| T-01-03 | Denial of Service | AttackSimulator worker threads | mitigate | Only one attack can run at a time (st.session_state.attack_running gate disables the launch button during active attacks). AttackSimulator uses bounded thread counts (5 threads for SQL injection per ATTACK_CONFIG). ngrok free tier has built-in rate limiting (40 connections/min). |
| T-01-04 | Information Disclosure | ngrok tunnel | accept | The attack panel is a public demo page with no secrets. The only exposed information is the attack panel UI itself. Hardcoded Docker credentials (root/rescue123) are never transmitted over ngrok -- they are only used for SSH rescue in Phase 3 (separate app on port 8501). |
| T-01-05 | Elevation of Privilege | Streamlit attack panel | accept | No authentication or authorization exists -- this is intentional per EXT-03 (no login required for classroom demo). The attack panel has no access to sensitive operations (SSH rescue, email sending, report generation -- all in the separate monitoring dashboard). |
| T-01-SC | Tampering | pip installs (ngrok via download) | mitigate | ngrok is downloaded directly from https://ngrok.com/download (official source). The launcher script checks for ngrok binary presence. No npm/pip/cargo package install for ngrok -- user downloads the binary. |

## Security Notes

- This is a classroom demonstration tool. The threat model is scoped to preventing abuse in a controlled classroom environment, NOT production hardening.
- The attack panel only targets localhost Docker nginx -- it CANNOT be used to attack external targets regardless of user input.
- ngrok free tier limitations (40 connections/min, 1 online process) provide natural rate limiting.
- If the classroom has more than 40 students simultaneously accessing the panel, ngrok will throttle connections -- this is acceptable for the demo scenario.
</threat_model>
